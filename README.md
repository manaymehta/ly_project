# LY Project — Pharmaceutical Supply Chain Disruption Intelligence System

An AI-powered dashboard that detects supply chain disruptions, predicts hospital stockout risk, and recommends emergency procurement plans for human review.

---

## Prerequisites

- [Miniconda or Anaconda](https://docs.conda.io/en/latest/miniconda.html)
- [Neo4j Desktop](https://neo4j.com/download/) (v5+)
- [Node.js](https://nodejs.org/) (v18+)
- A [Google AI Studio](https://aistudio.google.com/) account (free — for Gemini API key)

---

## Setup (run all commands from the project root)

### 1. Clone the repo

```bash
git clone <repo-url>
cd LY_Project
```

---

### 2. Configure environment variables

Copy `.env.example` to `.env` and fill in your values before anything else:

```bash
cp .env.example .env
```

Edit `.env`:
```
NEO4J_URI=neo4j://127.0.0.1:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=your_neo4j_password
GEMINI_API_KEY=your_google_ai_studio_key
```

Get your Gemini API key at: https://aistudio.google.com/app/apikey

---

### 3. Set up the Python environment

```bash
conda create -n ml_env python=3.12
conda activate ml_env
pip install -r requirements.txt
```

This installs everything including PyTorch (CPU build) and PyTorch Geometric.

---

### 4. Set up Neo4j

1. Open Neo4j Desktop → Create a new project → Add a local database
2. Set the password you used in `.env`
3. Start the database
4. Run the seeding script to populate all nodes and relationships from the CSV files:

```bash
conda activate ml_env
python scripts/seed_neo4j.py
```

Verify the graph looks correct:
```bash
python scripts/neo4j_audit.py
```

---

### 5. Train Prophet models

Trains 200 demand forecasting models (one per hospital-drug pair). Takes 2–5 minutes.

```bash
conda activate ml_env
python scripts/train_prophet_models.py
```

Models are saved to `ml/models/prophet_models/`.

---

### 6. Run GNN vulnerability scoring

Computes structural risk scores for every node and writes them to Neo4j.

```bash
conda activate ml_env
python ml/gnn_centrality.py
```

---

### 7. Start the backend

```bash
conda activate ml_env
uvicorn dashboard_api:app --reload --port 8000
```

---

### 8. Start the frontend

```bash
cd frontend
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173)

```bash

```

## Troubleshooting

**`NEO4J_URI` / `GEMINI_API_KEY` not found** — Make sure `.env` exists in the project root and is filled in. Do not rename `.env.example` without copying it.

**Neo4j connection refused** — Make sure the database is started in Neo4j Desktop before running any script.

**Prophet model not found** — Run `python scripts/train_prophet_models.py` from the project root first.

**Gemini 500 error** — API hiccup. The pipeline will retry automatically (up to 2 times). If all retries fail, a Retry button appears in the dashboard.

**Frontend blank page** — Make sure the backend is running on port 8000 before opening the frontend.

**VS Code shows `Cannot find module neo4j`** — This is a false warning caused by VS Code using the wrong Python interpreter. Select `ml_env` as your interpreter in VS Code (`Ctrl+Shift+P` → `Python: Select Interpreter`).
